# SymCascade：缓存状态感知的双路判别分流级联推理调度器

**设计文档（重构版）**
**日期**：2026-06-27
**目标**：会议投稿（会场未锁定，时间表仅作参考）
**重构依据**：原方案 F1-F6 可行性疑点验证 + I1-I4 创新性修改

---

## 1. 背景与定位

### 1.1 问题与动机

在云-边-端三层异构部署中，LLM Agent 推理面临成本-延迟-质量的多目标权衡。现有级联路由（FrugalGPT、RouteLLM、AutoMix、C3PO 等）的"低成本层"均为小 LLM，存在两个未被覆盖的空白：

1. **无人将符号规划器作为级联的确定性求解后端**——现有级联/推测式工作的低成本层全部是小 LLM，无一人引入符号规划器（PDDL + Fast Downward）。
2. **无人将语义缓存与符号规划结合**——缓存的不只是最终答案，而是符号规划器产出的 PDDL 骨架，命中时用符号搜索器做受约束重规划。

### 1.2 重构后的一句话定位

在云-边-端三层部署下，引入**缓存状态感知的双路判别分流级联**：在线 ROI 判别器依据 query 与当前缓存状态将请求分流到符号链（档1骨架重规划/档2生成PDDL求解）或神经链（档3 Gemma置信度/档4云端兜底），让多数请求在抵达云端前被拦截，实现降云端调用、降延迟、降成本的多目标 Pareto 改进。

### 1.3 与原方案的关键差异

| 维度 | 原方案 | 重构后 |
|------|--------|--------|
| 管线结构 | 单链逐层升级（L0缓存→L1 Gemma→L2 PDDL缓存→L3 LLM生成PDDL→L4云端） | 双路判别分流（D判别器→档1/2符号链 或 档3/4神经链） |
| 主创新1 | C1 符号规划器作低成本层 | **I1 双路判别分流 + 缓存状态感知在线 ROI 判别器** |
| 主创新2 | C2 PDDL骨架缓存 | C2 保留并强化（与 I1 耦合，域内两阶段验证） |
| C1 定位 | 独立创新点 | 降级为符号链求解后端（支撑性） |
| C3 定位 | 独立创新点（同源置信度门控） | 降级为消融变量 |
| C4 定位 | 拓展 ICML2025 级联最优性定理 | **I4 在线级联 regret 界**（原定理因结构变化失效） |

## 2. 核心创新点

### I1：双路判别分流 + 缓存状态感知在线 ROI 判别器（主创新1）

**机制**：缓存未命中时，由在线 ROI 判别器依据 query embedding 与当前缓存状态特征，将请求分流到三档：
- **档1（纯符号，零 LLM）**：缓存有匹配骨架 → 受约束重规划
- **档2（符号+神经，1 次边缘 LLM）**：可形式化但无骨架 → LLM 生成 PDDL + FD 求解
- **档3（纯神经）**：不可形式化 → Gemma 置信度门控 / 云端兜底

判别器输入 = query embedding（BGE-M3）⊕ 缓存状态特征（top-k 骨架相似度、缓存规模、历史命中率）。输出 = 三档 ROI 预测集，用 conformal prediction 给 1-α 覆盖率保证。在线学习：每 N 请求重训（滑动窗口 + EWMA 平滑应对 concept drift）。

**档与层级的对应**：档1=L1（骨架重规划），档2=L2（生成PDDL+求解），档3=神经链（含 L3 Gemma 置信度门控 + L4 云端兜底，档内再升级）。判别器选档后，档内有独立的升级链。

**为什么这是创新**：
- 判别器不判"二元可形式化性"（在 ALFWorld/HotpotQA 上 trivial），而是判"给定当前缓存状态，走哪档 ROI 最高"——缓存状态使问题非平凡（同 query 在缓存增长前后最优档会变）。
- 判别器与 C2 缓存耦合，形成**自成长调度器**：缓存越满→档1命中率越高→云端调用率单调下降。这条"缓存规模 vs 云端调用率"曲线是 I1 的可视化卖点。
- 回避了静态判别器的标签漂移问题：标签本就是"当前缓存状态下的最优档"，缓存状态作为判别器输入特征而非固定标签。

**文献空白**：现有级联/路由工作（FrugalGPT、RouteLLM、AutoMix、C3PO）均为单链逐层升级或单次路由，无"双路判别分流 + 缓存状态感知在线 ROI"机制。APC 缓存 plan template 但用小 LLM 适配，不与判别器耦合。

### C2：PDDL 骨架缓存 + 受约束重规划（主创新2，域内两阶段验证）

**机制**：语义缓存不只存最终答案，还缓存符号规划器产出的 PDDL plan skeleton（带前置条件/效果注解的动作序列模板）。命中时由 Fast Downward 在骨架前缀上做受约束重规划（fast replanning / LAMA-style restart from cached prefix）。

**两阶段验证策略**（解决原方案在 ALFWorld 上泛化价值不足的问题）：
- **阶段1（独立 ALFWorld）**：证"受约束重规划"——同骨架不同初始状态（物体在不同房间），重规划只需调 goto 目标位置，避免重新规划全 plan。
- **阶段2（混合负载）**：证"语义 embedding 模糊匹配"——混合流里 GAIA/GSM8K 可形式化子集经档2生成的骨架，与 ALFWorld 骨架跨任务类型复用，embedding 模糊匹配才有用武之地。

**不碰跨域迁移**：Blocksworld 的 stack 与 ALFWorld 的 put 动作语义不同，强行迁移 reviewer 会质疑。域内迁移（ALFWorld 内 pick_clean↔pick_heat 骨架同形）可行。

**文献空白**：
- APC（NeurIPS 2025）缓存 plan template 但用小 LLM 适配，不是符号搜索重规划。
- LTM-STM（MDPI 2024）缓存 PDDL skeleton 但领域特定，用 STM 修改而非符号搜索复用。
- "语义 embedding 模糊匹配 → 命中 PDDL 骨架 → 符号搜索器受约束重规划 + 判别器缓存状态感知"这一组合，文献中未见。

**自研内容**：
1. PDDL 骨架抽取器——把已解 plan 泛化为可复用 skeleton
2. 骨架语义匹配评估器——GPTCache 插槽无 PDDL 专用实现，需自写结构化距离度量（动作序列编辑距离 + 谓词图结构相似度）
3. 受约束重规划器——命中骨架时用 Fast Downward 在骨架前缀上做 fast replanning
4. 命中/重规划决策逻辑

### C1：符号规划器作求解后端（支撑性，非独立创新点）

**机制**：PDDL + Fast Downward/LAMA 作为档1（受约束重规划）与档2（LLM 生成 PDDL 后求解）的符号求解后端。

**定位变化**：原方案 C1 为独立创新点（"符号规划器首次作级联低成本层"）。重构后 C1 降级为符号链实现，其文献空白论证需重新表述为"符号规划器首次作判别分流后的确定性求解后端"。

**价值**：符号规划器具有低成本、可验证、确定性特性——CPU 上运行零 GPU 占用，求解结果可经 VAL 形式化校验，对可形式化子问题给出最优解。

### C3：同源置信度门控（消融变量，非独立创新点）

**机制**：利用 Gemma 4 12B 与 Gemini 3 的同源性，用 vLLM 的 logprobs 计算置信度，叠加 MAPIE conformal prediction 给覆盖率保证，作为档3神经链内部的升级门控（Gemma→云端）。

**定位变化**：原方案 C3 为独立创新点（"同源置信度门控"）。重构后降级为消融变量。

**F3 风险与处理**：Gemma 4 由 Gemini 3 研究构建，更可能是蒸馏/知识转移，蒸馏倾向于"同源同错"——学生学到老师错误模式。若 Gemma 高置信地错，Gemini 也可能错，则门控失效。这是结构性风险，故 C3 不作为独立创新点押宝，而在消融中验证 Gemma-Gemini（同源）vs Llama-Gemini（异构）。若消融发现同源红利显著，可回升为创新点；若证伪，论文不受损。

**conformal 保证语义**（F6 修正）：
- 判别器处：conformal 保证"ROI 预测集以 1-α 概率包含真实最优档"
- C3 神经链处：conformal 保证"不升级集以 1-α 概率不含需升级的 query"
- 均为 split conformal 标准用法，覆盖率≠质量保证的逻辑跳跃已修正

### I4：在线级联 regret 界（理论项，统一 I1/C2/C3）

**机制**：证明 SymCascade 相比最优离线策略（已知所有 query 真实 ROI，完美分流+升级）的 regret 上界：

```
regret = O(ε × T × (1 - h(t)) × α)
```

- ε：判别器 ROI 估计误差
- h(t)：缓存命中率（随时间单调增长）
- α：conformal 失败率
- T：请求数

**统一性**：界中三项分别对应 I1（ε）/ C2（h(t)）/ C3（α），把三个组件统一到同一理论界。

**为什么是 regret 界而非最优性定理**：I1 重构后级联结构从"单链逐层升级"变"双路判别+在线ROI"，原 ICML2025 贪心最优性定理完全不适用（缓存命中是跳层非升级、判别器在线学习使 ROI 时变）。regret 框架天然契合在线学习场景。

**证明工具**：online learning 工具，目标 ε-最优界。若严格界困难，降级为 ε-最优界或实验性 regret 验证。

## 3. 系统架构与数据流

### 3.1 级联管线（重构）

```
请求到达
   │
   ▼
[L0: 语义缓存层] ──命中──► 直接返回 (零 LLM 调用)
   │未命中
   ▼
[D: 判别器] (在线 ROI, 输入: query emb ⊕ 缓存状态特征)
   │
   ├──档1: 缓存有匹配骨架──► [L1: PDDL骨架受约束重规划] (CPU, 零LLM)
   │                            ├──成功──► 返回 + 更新判别器
   │                            └──失败──► 回退档2
   │
   ├──档2: 可形式化无骨架──► [L2: LLM生成PDDL + FD求解] (1次边缘LLM + CPU)
   │                            ├──成功──► 返回 + 抽取骨架存缓存 + 更新判别器
   │                            └──失败──► 回退档3
   │
   └──档3: 不可形式化──► [L3: Gemma置信度门控] (边缘GPU, thinking=False)
                                ├──高置信(conformal)──► Gemma答
                                └──低置信──► [L4: 云端Gemini兜底]
```

### 3.2 各层成本-质量特性

| 层级 | 成本 | 延迟 | 质量 | LLM 调用 | GPU |
|------|------|------|------|---------|-----|
| L0 语义缓存 | ~0 | ~0 (内存查询) | 同缓存原值 | 0 | 无 |
| D 判别器 | 极低 | 极低 | - | 0 | 无/极低 |
| L1 骨架重规划 | 低 | 低 (CPU 搜索) | 高 (确定性最优) | 0 | 无(CPU) |
| L2 LLM+符号规划 | 中 | 中 | 高 | 1 (生成 PDDL) | 边缘 GPU + CPU |
| L3 Gemma 边缘 | 0 (本地) | 低 | 中-高(置信度高时) | 1 | 边缘 GPU |
| L4 云端 Gemini | 高 | 高 | 最高 | 1 | 云端 |

### 3.3 数据流关键约束

- **判别器输入**：query embedding（BGE-M3）⊕ 缓存状态特征（top-k 骨架相似度、缓存规模、历史命中率）
- **判别器输出**：三档 ROI 预测集，conformal 给 1-α 覆盖率
- **在线学习**：每 N 请求重训，滑动窗口 + EWMA 平滑应对 concept drift，drift 检测触发回退静态判别器
- **保守偏可形式化**：误判"不可形式化"→实可形式化代价高（错过符号最优解），故判别器阈值偏向档1/档2
- **回退链**：档1失败→档2，档2失败→档3，保证不丢请求
- **Gemma thinking 约束**：低成本层强制 `enable_thinking=False`，仅难度预估可选开启

### 4.1 Benchmark 选型与验证目标

| Benchmark | 角色 | 验证目标 |
|-----------|------|---------|
| **ALFWorld**（独立） | 符号规划主载体 | C2 受约束重规划（同骨架不同初始状态） |
| **HotpotQA/GSM8K**（独立） | 神经链验证 | C3 神经链门控 |

### 4.2 混合负载构造

混合负载是 I1 判别器与 C2 模糊匹配成立的前提。构造需有依据，避免 cherry-pick 质疑：

- **到达率/难度分布**：参考公开云-边-端部署 trace，或基于 ALFWorld+HotpotQA+GAIA+GSM8K 加权采样
- **任务类型比例**：报告多种配比下的敏感性分析（如可形式化:不可形式化 = 3:7 / 5:5 / 7:3）
- **冷启动处理**：明确判别器在线学习的起始状态与预热期

### 4.3 基线方案（分层对照）

| 基线 | 机制 | 主对照场景 |
|------|------|-----------|
| B1 纯云端 | 所有请求送 Gemini 3 | 上界质量（全部场景） |
| B2 纯边缘 | 所有请求送 Gemma 4 12B | 下界成本（全部场景） |
| B3 FrugalGPT | 小→大 LLM 级联+蒸馏缓存 | 混合负载 + HotpotQA/GSM8K |
| B4 RouteLLM | 学习型单次路由 | 混合负载 + HotpotQA/GSM8K |
| B6 APC | Agentic Plan Caching（小 LLM 适配） | ALFWorld（C2 直接对照） |
| B7 LLM-DP | LLM 生成 PDDL + FD 求解（96%） | ALFWorld（C1 符号链基线） |

**B6 APC 是原方案缺失的关键基线**——C2 声称"符号重规划优于 APC 的小 LLM 适配"，无 APC 基线则此声称无法证实。APC 原为 NL plan 适配，跑 ALFWorld 需明确改造边界，报告改造前后对比。

### 4.5 实验分组

| 实验 | 目的 | 主要指标 |
|------|------|---------|
| E1 级联效果 | 双路判别分流是否降本保质 | 云端调用↓%、质量保持% |
| E2 成本-质量 Pareto | 不同预算下最优质量前沿 | Pareto 前沿图 + HV |
| E3 延迟分布 | 端到端延迟 | p50/p99 + 首 token 延迟 CDF |
| E4 消融 | I1/C2/C3 各模块贡献 | 逐模块质量/成本/延迟变化 |
| E5 难度/负载可扩展 | 不同难度分布、并发10→1000 | 各档命中率、吞吐曲线 |
| E6 自成长曲线 | 缓存规模 vs 云端调用率 | 单调下降曲线（I1 卖点） |
| E7 判别器分析 | 各档命中率/分流准确率/在线学习稳定性 | Sankey 图、drift 检测 |

### 4.6 结果评判方法

**第一层：客观指标优先**
- ALFWorld：任务成功率（程序化检查，无 judge）
- HotpotQA/GSM8K：EM/F1（与 ground truth 比对）
- GAIA：exact-match（避免 judge 保证可复现）

**第二层：Pareto 多目标评判**
- 超体积 HV：bootstrap 95% CI 比较支配显著性（参考点取 nadir）
- Pareto 支配判断 + ε-指示子
- 呈现：Pareto 前沿图 + 采样数值表

**第三层：统计显著性检验**
- ≥10 次独立运行（不同随机种子），报告 mean±std
- Wilcoxon signed-rank test（配对非参数）
- Bootstrap 置信区间（B=1000-5000）
- 多重比较校正：Holm 校正（比 Bonferroni 温和）
- 效应量：Cliff's delta

### 5.2 按创新点拆解

**I1 双路判别分流（核心自研）**
- 判别器：GBDT 或小 MLP，输入 query emb ⊕ 缓存状态特征
- 在线学习框架：滑动窗口重训 + EWMA + drift 检测
- conformal 集成：MAPIE 给 ROI 预测集
- 工作量：中高（新增主创新）

**C2 PDDL 骨架缓存+符号重规划（核心自研）**
- 起点：GPTCache 模块化缓存键插槽
- 必须从 0 写：
  1. PDDL 骨架抽取器
  2. 骨架语义匹配评估器（GPTCache 插槽无 PDDL 专用实现）
  3. 受约束重规划器
  4. 命中/重规划决策逻辑

## 7. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| 判别器在线学习不稳定（concept drift） | I1 价值不显著 | 滑动窗口重训 + EWMA 平滑 + drift 检测触发回退静态判别器 |
| 混合负载构造被质疑 cherry-pick | I1 实验可信度 | 基于公开 trace 或多 benchmark 加权采样，报告多种配比敏感性分析 |
| C3 同源红利被证伪 | C3 失效 | 已降级为消融变量，证伪不影响主创新 I1+C2 |
| I4 regret 界证明困难 | 理论项落空 | 降级为 ε-最优界或实验性 regret 验证 |
| APC 改造跑 ALFWorld 公平性 | B6 基线不公平 | 明确 APC 改造边界（NL 适配→PDDL 适配），报告改造前后对比 |
| 符号规划器对非 ALFWorld 任务形式化困难 | C1/C2 适用范围受限 | ALFWorld 为主载体；混合负载上 C2 模糊匹配验证跨任务类型复用 |

| 骨架缓存命中率低 | C2 价值不显著 | 预实验在 ALFWorld 6 类任务上测命中率；混合负载验证跨类型复用 |
| 级联多次尝试累积延迟 | 端到端延迟增加 | 判别器早停 + 缓存命中短路 + 回退链限时 |

## 9. 决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 管线结构 | 双路判别分流（I1） | 原单链层序次优，符号层应在神经层前；判别器解决可形式化分流 |
| 判别目标 | 缓存状态感知在线 ROI | 回避二元判别 trivial、ROI 冷启动难、三档标签漂移三硬伤 |
| 评测负载 | 混合主+独立辅 | 混合证 I1 判别器+C2 模糊匹配；独立证各链质量不劣于专精基线 |
| C2 载体 | 域内两阶段 | ALFWorld 证重规划，混合负载证模糊匹配；不碰跨域迁移 |
| C3 定位 | 降级为消融变量 | 同源红利可能被证伪，不押宝为核心卖点 |
| C4 定位 | I4 在线 regret 界 | 原定理因 I1 结构变化失效；regret 框架契合在线学习且统一三组件 |
| 基线体系 | 分层对照 + 补 APC/LLM-DP | 解决 FrugalGPT/RouteLLM 在 ALFWorld 不公平；APC 是 C2 直接对照缺失 |
| 创新点结构 | I1+C2 主创新，C1 支撑，C3 消融，I4 理论 | I1 重构后 C1/C3 边际收益低，降级集中火力 |
